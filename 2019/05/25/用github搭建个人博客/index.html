##用github搭建个人博客##

在网上看了几个教程，写的不错，但是在搭建的过程中仍然遇到了个别问题，故记录以下，仅供参考。

###第一步：搭建Github Pages###

  1. 注册GitHub。不多说了。
  2. 创建仓库。需要注意Repository name一定要设置为注册的github名.github.io，这样就可以通过它访问博客主页。我已经创建过了，所以有红色的提示。这说明每个帐号只能有一个仓库来存放个人主页，而且仓库的名字必须是username/username.github.io，这是特殊的命名约定。

  1. 开启Github Pages并选择主题。进入创建好的仓库，点击Settings,向下划找到Github Pages这一栏，如果仓库名字设置对了，就会显示Your site is published at…，可以点击Choose a theme进入选择主题的界面。

###第二步：配置系统环境###

  1. 安装Node.js

下载Node.js

安装参考

我在安装时就遇到了问题，下载了zip文件却不能安装，都怪自己无知，不知道啥是二进制文件，然后又下载了msi格式的才安装上。

  1. 安装Git

下载地址

一路点击Next就行了.

  1. 安装Hexo

首先在磁盘上选择一个位置新建一个文件夹以存放生成的Hexo文件，该文件存放博客的内容等，以D:/blog为例。进入blog文件夹内右键选择Git Bash Here，在命令行里分别输入以下命令。

npm install hexo-cli -g
hexo init
npm install
hexo g
hexo s

几个常用Hexo命令：
hexo generate (hexo g) 生成静态文件，会在当前目录下生成一个新的叫做public的文件夹；
hexo server (hexo s) 启动本地web服务，用于博客的预览，可以通过http://localhost:4000/ 查看；
hexo deploy (hexo d) 部署播客到远端（比如github, heroku等平台）；
hexo new “postName” 新建文章；
hexo new page “pageName” 新建页面；

常用组合
hexo d -g #生成部署
hexo s -g #生成预览

###第三步：添加主题###
Hexo博客主题地址
进入Hexo网站后点击主题进入，这里有250多个主题可以选择，找到喜欢的可以点击图片预览，点图片下面的名称则可进入github，查看源代码以及使用说明。



以配置主题black-blue为例：

  1. 安装

    hexo clean
    cd themes && git clone git clone https://github.com/maochunguang/black-blue

  2. 启动
    返回到Hexo目录下，打开_config.yml文件，找到theme属性，修改为theme: black-blue

  3. 更新主题

    cd themes/black-blue
    git pull
    hexo g
    hexo s

此时刷新http://localhost:4000/页面就能看到新的主题了。

###第四步：配置SSH key###

1.设置SSH keys
在git bash中依次输入以下命令，生成SSH key

cd ~/.ssh
ls
mkdir key_backup
cp id_rsa* key_backup
rm id_rsa*  
#以上三步为备份和移除原来的SSH key设置
ssh-keygen -t rsa -C "jpdong00@gmail.com" #生成新的key文件,邮箱地址填你的Github地址
#Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>
#接下来会让你输入密码

2.添加SSH Key到Github
首先到Github首页,点击Settings进入设置，找到SSH and GPG keys，点击New SSH keys。


接着(开启查看隐藏文件)找到C:\Users\用户名.ssh id_rsa.pub文件，以文本方式打开，打开之后全部复制到key中

做到这里就可以打开git bash输入以下命令测试是否配置成功:

ssh -T git@github.com

然后设置你的账号信息:

git config –global user.name “你的名字”
git config –global user.email “邮箱@邮箱.com” #github邮箱

###第五步：部署到github###
1.再次打开Hexo目录下的_config.yml文件，修改deploy属性

deploy:
type: git
repo: git@github.com:jpdong/jpdong.github.io.git
branch: master

2.安装一个扩展

npm install hexo-deployer-git –save

3.进行部署：
方式一：在命令行中输入以下命令即可完成部署。

hexo d

方式二：不幸的是，虽然方式一简单方便，但是偶尔会有莫名其妙的问题出现，可以使用git命令来完成部署的工作。

首先，将之前创建的仓库克隆到本地，新建一个目录叫做.deploy的文件夹来存放克隆的代码。

cd d:/hexo/blog
git clone https://github.com/jpdong00/jpdong00.github.io.git .deploy/jpdong00.github.io

然后创建一个deploy脚本文件，执行部署工作。 脚本可以保存为deploy.sh，第一次执行时要输入Github账户的用户名及密码，按照提示操作即可。

hexo generate
cp -R public/* .deploy/jpdong00.github.io
cd .deploy/jiji262.github.io
git add .
git commit -m “update”
git push origin master

代码的含义如下：hexo generate生成public文件夹下的新内容，然后将其拷贝至jpdong00.github.io的git目录下，再使用git commit命令提交代码到jpdong00.github.io这个仓库的master branch上。

这时再刷新jpdong00.github.io就可以看到搭建成功的博客了。

###第六步：完善博客内容###
到第五步为止，一个简单的博客就搭建完成了，接下来就是更细致的配置了，未完待续。
